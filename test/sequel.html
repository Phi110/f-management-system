テストお疲れさまでした！！！


予想問題を尽く外して悲しい気持ちになっていますが、とりあえずテスト並びに後期半年間お疲れさまでした！ C++ の問題が想定より難しくて、解きながらみんなごめんなさいって思ってました笑

最後に、今後もプログラミングをしていると沢山出てくるであろう "クラス" について、 身近に感じてもらえるのではないかという例を思いついたので、説明したいと思います。


.size() .length

と聞くと、文字列とかリストとかの長さを測るあれねって感じで、皆さん馴染み深いと思います。実はこれらも広い意味で言うとクラスが関係しているんです。 例えば、 

string name = "toby";
cout << name.size() << endl;
というコードを書くと、 
4
と出力されます。これはいつも通りだと思います。

ここで、String クラスを以下のように定義します。 
class String {
public:
    int size_str;     // メンバ変数 size_str
    String (string str0) {     // コンストラクタ
        size_str = str0.size();
    }
    int size () {     // メンバ関数 size
        return size_str;
    }
};
( string 型と .size() を使っているのは目を瞑ってください。 )

Stringクラスのインスタンス name を生成して、初期値に "toby" を渡します。
name のメンバ関数 (Python ではメソッド) の size () を呼び出すと、 
int main() {
    String name {"toby"};
    cout << name.size() << endl;
}
のようになり、最初のコードとほとんど同じになっているのがわかります。
出力結果は当然 4 になります。 
4
このように、私たちは日常的に メンバ関数 (メソッド) を呼び出しているのです！

size () のような関数を毎回書いていては、コードが長くなって実装も大変になってしまいますが、 モジュールの形でインポートすることで、複雑な処理も簡単に書きやすくなります。
これがいわゆる オブジェクト指向 というものです。

オブジェクト指向 を意識してコーディングが出来ると、より実用的なコードに近づくと思います。
更なるスキルアップのために、これからも一緒に頑張りましょう！！
